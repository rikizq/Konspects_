Для работы с API на языке Python существуют несколько модулей и фреймворков. Среди них — библиотеки Requests, Aiohttp, HTTPX и фреймворк Django REST Framework (DRF)
## Requests

**Библиотека для работы с HTTP-запросами**. Обеспечивает интерфейс для выполнения операций с REST API, загрузки данных с веб-страниц, а также отправки файлов и авторизации. [
Особенности:

- Поддержка всех методов HTTP — GET, POST, PUT, DELETE, PATCH, HEAD, OPTIONS.
- Автоматическое декодирование контента.
- Поддержка сессий, прокси, аутентификации.
- По умолчанию поддерживает SSL, автоматически проверяет сертификаты.
Объект response содержит данные, возвращаемые сервером, в том числе контент, код состояния и заголовки. Некоторые атрибуты ответа:
status_code
код состояния HTTP (например, 200 для успеха);
text
содержимое ответа в виде текста;
json()
декодирует JSON-ответ в объект Python.

## Aiohttp
Асинхронная библиотека для работы с HTTP-клиентом и сервером на Python. Построена на основе asyncio. 

**Особенности**:

Поддерживает создание как простых HTTP-клиентов для взаимодействия с внешними API, так и полноценных веб-серверов с поддержкой middleware, маршрутизации и шаблонизации.
Основной класс для выполнения HTTP-запросов ClientSession который управляет пулом соединений и обеспечивает эффективное пере использование соединений.
В библиотеке есть объектClientResponse который позволяет читать контент ответа и его статус-код.

## httpx
Модуль для работы с HTTP-запросами. Позволяет отправлять синхронные и асинхронные HTTP-запросы. 

Особенности:
- Поддерживает HTTP/1.1, HTTP/2 и WebSocket.
- Имеет функции для настройки запросов, например, встроенные таймауты и retry-механизмы.
- Поддерживает потоковую обработку запросов, что позволяет работать с большими объёмами данных. [
- В модуле есть объект httpx.Response(), который представляет ответ от HTTP-запроса и содержит данные. 

Некоторые методы объекта:
##### status_code
возвращает код состояния HTTP; 
##### text
возвращает тело ответа в виде строки; 
##### json()
анализирует тело ответа как JSON, возвращает словарь или список. 
## Основы REST API для Python-разработчиков

REST API (Representational State Transfer Application Programming Interface) - это архитектурный стиль для создания веб-сервисов, который использует HTTP-протокол для обмена данными. Для Python-разработчиков понимание REST API является интересным навыком, позволяющим взаимодействовать практически с любым современным веб-сервисом.

Прежде чем погружаться в код, важно разобраться в ключевых концепциях:
- Endpoints — URL-адреса, по которым доступны различные функции API
- HTTP-методы — GET, POST, PUT, DELETE, определяющие тип операции
- Headers — метаданные запроса, включающие информацию об аутентификации
- **Параметры запроса** — данные, передаваемые через URL или тело запроса
- **Статус-коды** — числовые индикаторы результата запроса (200, 404, 500 и т.д.)

REST API обычно возвращает данные в формате JSON (JavaScript Object Notation), который Python отлично обрабатывает благодаря встроенной библиотеке json.

| **HTTP-метод** | **Назначение**                        | **Пример использования**              |
| -------------- | ------------------------------------- | ------------------------------------- |
| GET            | Получение данных                      | Запрос информации о пользователе      |
| POST           | Создание новых данных                 | Регистрация нового пользователя       |
| PUT            | Полное обновление существующих данных | Обновление всего профиля пользователя |
| PATCH          | Частичное обновление данных           | Изменение только имени пользователя   |
| DELETE         | Удаление данных                       | Удаление учетной записи пользователя  |
## Коды состояния
После обработки запроса сервер возвращает ответ со статус-кодом — трёхзначным числом, которое указывает на результат обработки запроса. Некоторые категории кодов:

- 2xx — успешные коды, указывают, что запрос был успешно получен, понят и обработан сервером. Например, 200 OK — стандартный успешный ответ для большинства запросов.
- 3xx — коды перенаправления, указывают, что клиент должен выполнить дополнительные действия для завершения запроса. Например, 301 Moved Permanently — ресурс был окончательно перемещён по новому URI.
- 4xx — клиентские ошибки, указывают, что запрос содержит ошибку или не может быть выполнен. Например, 400 Bad Request — неверный синтаксис запроса или невалидные параметры.

## Разница между front-end и back-end

#### Front-end

Front-end – это та часть веб-приложения, с которой взаимодействует пользователь. Это всё, что с чем мы взаимодействуем на веб-странице: кнопки, текстовые поля, изображения, навигация и т.д. Front-end разрабатывается с использованием таких технологий, как HTML, CSS и JavaScript. Он отвечает за отображение данных и обработку пользовательского ввода.

#### Back-end

Back-end – это "невидимая" часть веб-приложения, которая работает на сервере. Он отвечает за обработку данных, взаимодействие с базой данных, аутентификацию пользователей и другие задачи, которые происходят "за кулисами". Back-end разрабатывается с использованием серверных языков программирования, таких как Python, Java, PHP и других.
