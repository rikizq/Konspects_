### Введение

Процесс удаленного подключения к другой системе на oc Windows (в нашем случае Windows 10), можно осуществить несколькими различными способами, одним из которых является подключение через встроенную утилиту подключения Windows 10, реализующую протокол RDP.
### История RDP

Протокол RDP (Remote Desktop Protocol) был разработан компанией Microsoft для удалённого управления компьютерами. Впервые он появился в 1998 году в Windows NT 4.0 Terminal Server Edition. В дальнейшем RDP стал частью Windows и получил поддержку шифрования, передачи звука, принтеров и других устройств. Сейчас RDP является стандартным средством удалённого подключения в операционных системах Windows.

### Подключение к удалённому рабочему столу
1. Настройка удалённого компьютера

Открыть Параметры → Система → Удалённый рабочий стол.

Активировать параметр «Включить удалённый рабочий стол».

Проверить, что учётная запись, через которую будет выполняться вход, имеет установленный пароль.

2. Подключение к удалённому компьютеру

Нажать сочетание клавиш Win + R, ввести команду mstsc и нажать Enter.

В открывшемся окне программы «Подключение к удалённому рабочему столу» указать IP-адрес или имя удалённого компьютера.

Нажать кнопку «Подключиться».

После появления окна авторизации ввести имя пользователя и пароль учётной записи удалённого компьютера.

После успешной аутентификации откроется рабочий стол удалённого компьютера, и с ним можно будет работать как с локальным устройством. (Подключился к Алексею Репину)

## Важные нюансы и ограничения при работе с RDP
### Недостатки удалённого подключения:

1. Необходимость пароля
Для подключения по RDP учётная запись удалённого пользователя должна иметь пароль. Подключение к учётным записям без пароля по умолчанию запрещено из соображений безопасности. 

2. Одновременная работа пользователей
В Windows 10 и Windows 11 одновременно работать за одним компьютером через RDP нельзя. При подключении удалённого пользователя локальная сессия блокируется.

3. Права доступа
Для использования удалённого подключения пользователь должен иметь соответствующие права. В моём случае подключение выполнялось через учётную запись администратора, поэтому для входа требовался пароль администратора. 

4. Подключение через Интернет
По умолчанию RDP лучше использовать внутри локальной сети. Если необходимо подключаться через Интернет, рекомендуется использовать VPN. Это позволяет создать защищённый канал связи между устройствами и снижает риск несанкционированного доступа к удалённому компьютеру. Открывать порт RDP (3389) напрямую в Интернет не рекомендуется из-за возможных атак на систему.

### Полезные возможности RDP
Перед подключением можно открыть окно «Показать параметры» в программе mstsc.

1. Во вкладке «Локальные ресурсы» можно включить общий буфер обмена для копирования текста и файлов между компьютерами.

2. Во вкладке «Экран» можно настроить разрешение удалённого рабочего стола и использовать несколько мониторов.

3. Настройки подключения можно сохранить в файл и использовать повторно без повторного ввода параметров.
