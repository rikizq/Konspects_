#  Алгоритмы и структуры данных (DSA)

###  Основные понятия

- Data Structure (Структура данных) - это способ организации и хранения данных в памяти, который позволяет эффективно ими управлять (добавлять, искать, изменять).

- Algorithm (Алгоритм) - четкая последовательность действий для решения конкретной вычислительной задачи.
###  Цели изучения

- Эффективность: Написание кода, который работает быстрее и потребляет меньше оперативной памяти.

- Оптимизация: Правильный выбор инструмента (например, использование Хеш-таблицы вместо обычного списка) ускоряет программу в десятки и сотни раз.
###  Текстовый гайд (Programiz DSA)
Основной упор на интерактивное изучение и визуализацию структур:

- Анализ сложности ($Big O$): Оценка производительности кода и его масштабируемости.

- Линейные структуры:

- Стеки (Stack): Принцип LIFO (последним пришел - первым ушел).
  
- Очереди (Queue): Принцип FIFO (первым пришел - первым ушел).

- Связные списки (Linked Lists): Узлы, связанные ссылками.

- Нелинейные структуры:

- Деревья (Trees): Иерархические структуры (AVL, B-деревья).

- Графы (Graphs): Сложные связи между объектами и алгоритмы обхода (BFS/DFS).

- Алгоритмы:

- Различные виды сортировок (от простых до сложных типа Radix).

- Методы «Разделяй и властвуй» (Divide and Conquer).
###  Практическое применение
- Использование визуализаций для понимания логики работы структур.
- Реализация алгоритмов на языке программирования для закрепления теории.
- Подготовка базы для решения сложных задач в разработке.
