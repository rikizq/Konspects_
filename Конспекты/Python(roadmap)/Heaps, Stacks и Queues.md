## Stack

Stack (стек) - линейная структура данных, которая предоставляет доступ только к последнему добавленному элементу. Все операции выполняются на вершине стека.

Основные операции стека:

* `push` - добавление элемента;
* `pop` - удаление верхнего элемента;
* `peek` (`top`) - получение верхнего элемента без удаления.

Добавление и удаление элементов выполняются за O(1).

В Python стек обычно реализуется через `list`, поскольку список позволяет эффективно работать с концом последовательности.

Стек используется при вызове функций, работе рекурсии, обработке математических выражений, проверке баланса скобок и обходе графов в глубину (DFS).

## Queue

Queue (очередь) - линейная структура данных, в которой элементы обрабатываются в порядке их поступления. Добавление выполняется в конец очереди, а удаление - из начала.

Основные операции:

* `enqueue` - добавление элемента;
* `dequeue` - удаление первого элемента;
* `front` - получение первого элемента без удаления.

Для реализации очередей в Python обычно используется `collections.deque`, так как эта структура обеспечивает O(1) для добавления и удаления элементов с обоих концов.

Очереди применяются при обработке запросов, планировании задач, работе с буферами данных и реализации алгоритма поиска в ширину (BFS).
## Heap

Heap (куча) - структура данных на основе полного бинарного дерева, которая обычно хранится в виде массива.

В Python стандартной реализацией является Min Heap. В такой структуре значение каждого родительского узла меньше или равно значениям его потомков. Благодаря этому минимальный элемент всегда находится в корне кучи.

Основные операции:

* вставка элемента - O(log n);
* извлечение минимального элемента - O(log n);
* получение минимального элемента - O(1);
* построение кучи - O(n).

После добавления или удаления элемента выполняется перестройка структуры для сохранения свойств кучи.

Для работы с кучами в Python используется модуль `heapq`. Он предоставляет функции для создания кучи, добавления элементов и извлечения минимального значения.
## Отличия структур

Stack и Queue являются линейными структурами данных. Разница между ними заключается в порядке обработки элементов. В стеке первым извлекается последний добавленный элемент, а в очереди - первый добавленный.

Heap отличается тем, что не хранит элементы в порядке добавления. Его задача - поддерживать специальное свойство дерева, которое позволяет быстро получать минимальный элемент. За счёт этого кучи часто используются в алгоритмах, где необходимо постоянно выбирать элемент с наивысшим или наименьшим приоритетом.
