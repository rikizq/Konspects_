## Слова прямо в коде (массив/список)

```python
words = ["яблоко", "груша", "машина", "дом"]
```

### Плюсы

1. Максимальная простота
        
2. Минимум зависимостей
        
3. Быстрота разработки
### Минусы

1. Плохая масштабируемость
	Большой список засоряет код
        
2. Сложность обновления
    Чтобы добавить слово, нужно менять код
        
3. Нарушение читаемости
    Логика игры смешивается с данными


## Отдельный Python-файл со списком слов

```python
words.py words = ["яблоко", "груша", "машина"]
```

```python
from words import words
```

### Плюсы

1. Чище основной код
    Игровая логика отделена от данных
        
2. Легко расширять
    Добавлять слова можно без изменения игры
        
3. Простота импорта
    Не требуется работа с файлами и вводом-выводом
        

### Минусы

1. Требует знания Python
нужно знать синтаксис

2. Жёсткая структура
формат ограничен кодом

3. Неудобно для больших списков
много слов для .py


## Код + файл .txt со словами

```txt
яблоко
груша
машина
```

```python
with open("words.txt", encoding="utf-8") as f:
	words = f.read().splitlines()
```
### Плюсы

1. Полное разделение логики и данных
код отвечает за игру, а файл за контент
        
2. Простота для пользователя
слова можно менять без знания .py
        
3. Хорошая масштабируемость
подходит для больших словарей

### Минусы

1. Нужна обработка ошибок
    Файл может отсутствовать или быть повреждён
        
2. Зависимость от внешнего файла
    Без words.txt игра не запустится
        
3. Чуть сложнее реализация
    Нужно читать файл, учитывать кодировку, формат
