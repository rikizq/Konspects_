# Инкапсуляция
Принцип объектно-ориентированного программирования (ООП), который заключается в скрытии внутреннего состояния объекта и предоставлении доступа к этому состоянию только через строго определённые методы.

Особенности:
- защитить данные от некорректного использования.
- скрыть сложную логику пользователь работает с простым интерфейсом, не зная деталей реализации.
- обеспечить гибкость изменений (можно менять внутреннюю логику, не ломая внешний код.
# Наследование
Механизм объектно-ориентированного программирования (ООП), позволяющий создавать новый класс на основе уже существующего.

Особенности:
- Дочерний класс автоматически получает все атрибуты и методы родительского класса, ему не нужно определять их заново.
- Может добавлять собственные особенности и модифицировать унаследованное поведение.
- Может переопределять методы родительского класса, создавать метод с таким же названием, но с другой реализацией.

# Полиморфизм
Это способность объектов разных классов реагировать на одинаковые методы или операции по-разному. Это позволяет использовать объекты разных типов через единый интерфейс, не беспокоясь о конкретном типе.
## Виды

В Python полиморфизм реализуется двумя способами:

1. Через наследование. От базового класса наследуются классы-потомки, которые реализуют у себя его методы. Сигнатура методов совпадает, и к ним может обращаться вызывающий код вне зависимости от класса объекта, с которым он работает.
2. Через «утиную типизацию» (duck typing). Если объект имеет нужные методы, то его тип не важен. Это основной подход в Python, не требует формального наследования.

## Сильные стороны полиморфизма:

- Гибкость кода — возможность работать с разными типами объектов через одинаковый интерфейс.
- Расширяемость — новые классы могут быть добавлены без изменения существующего кода.
- Упрощение кода — можно написать один метод вместо нескольких похожих.
