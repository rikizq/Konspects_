# Определение
Исключения — неотъемлемая часть разработки надежных приложений на Python. Их правильная обработка — не просто хорошая практика, а необходимость. Она помогает создавать программы, способные справляться с различными нештатными ситуациями без аварийного завершения.

### Как работают исключения в Python
В Питоне технология обработки исключений основана на передаче объектов исключения. Когда возникает ошибка, Python создает экземпляр класса-исключения, содержащий информацию о типе ошибки и ее контексте . Этот объект затем «пробрасывается» вверх по стеку вызовов.

Поиск обработчика начинается с текущей функции. Если в этой функции есть блок try-except, который обрабатывает тип этого исключения то выполнение переходит в блок except. Этот блок содержит код, который тем или иным образом обрабатывает ошибку: например, выводит сообщение пользователю, выполняет очистку ресурсов или пытается выполнить альтернативное действие.

Обработка исключений улучшает сопровождаемость кода. Его разделение на блоки try-except четко выделяет части программы, которые могут вызвать ошибки. Это упрощает отладку, что появятся новые ошибки. В итоге опыт пользователя улучшается, предоставляются информативные сообщения о багах, а программа может продолжать работу даже после их возникновения.

Если в текущей функции нет подходящего блока except, исключение пропускается вверх по стеку вызовов в вызывающую функцию. Этот процесс повторяется, пока не будет найден подходящий обработчик или пока исключение не достигнет вершины стека, что приводит к аварийному завершению программы с соответствующим сообщением.
